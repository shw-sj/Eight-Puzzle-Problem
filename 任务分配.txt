【成员A 补充 — 算法核心细节】

  1. 状态数据结构
     - 一维表示：[1,2,3,4,5,6,7,8,0]（方便哈希存入set）
     - 二维表示：[[1,2,3],[4,5,6],[7,8,0]]（方便移动操作）
     - 转换函数：to_1d() / to_2d()，搜索时统一用一维串作closed表key

  2. Node节点类
     - 属性：state, parent, g(n), h(n), f(n), action(上/下/左/右)
     - __lt__按f(n)排序，f相同时按h排序（heapq需要）

  3. 移动函数 get_neighbors(state)
     - 找到空格位置(0)
     - 生成上下左右四个方向的新状态
     - 边界检查：空格在第一行不能up、最后一行不能down等
     - 返回 [(new_state, action), ...]

  4. OPEN表（小顶堆）
     - import heapq
     - 压入：(f值, 节点id, Node)，弹出总是最小f
     - 用节点id防止Node不可比较时报错

  5. CLOSED表
     - set()存储 state_tuple（一维tuple），O(1)查找
     - 也存 {state_tuple: g_value} 字典，方便更新OPEN中更优路径

  6. 曼哈顿距离计算
     - 对每个数字1~8：找到cur位置和目标位置的坐标差绝对值之和
     - 注意：空格0不参与计算


【成员B 补充 — 对比实验具体方案】

  1. 逆序数可解性（详细步骤）
     - 将初始状态二维→一维，去掉0
     - 双重循环统计逆序对个数
     - 如果(初始逆序数%2) != (目标逆序数%2)，则不可解
     - 原理：每次移动空白格不改变逆序数的奇偶性

  2. 错位棋子数 h2
     - 统计a[i][j] != goal[i][j] 且 a[i][j] != 0 的个数
     - 优点：计算极快（O(1)）
     - 缺点：精度低，扩展节点多

  3. 欧几里得距离 h3
     - Σ √((x1-x2)² + (y1-y2)²)
     - 精度介于 h2 和 h1 之间
     - 计算稍慢（需要sqrt），但扩展节点比h2少

  4. 测试用例设计（建议至少8组）
     用例1-2：简单（2-5步可解，如只需滑动1-2个数字）
     用例3-4：中等（6-12步可解）
     用例5-6：困难（13-20步可解）
     用例7：极端（20+步可解，如经典最难布局）
     用例8：不可解样例

  5. 结果对比表格模板
     ┌──────┬──────────┬──────────┬──────────┬────────────┬──────────┐
     │ 用例  │ 启发式   │ 扩展节点  │ 最大OPEN │ 路径长度   │ 耗时(ms) │
     ├──────┼──────────┼──────────┼──────────┼────────────┼──────────┤
     │ 用例1 │ h1       │    xxx    │   xxx    │    xxx     │   xxx    │
     │      │ h2       │    xxx    │   xxx    │    xxx     │   xxx    │
     │      │ h3       │    xxx    │   xxx    │    xxx     │   xxx    │
     └──────┴──────────┴──────────┴──────────┴────────────┴──────────┘

  6. 算法改进方向（选1-2个）
     a) IDA*：设置depth_limit = f(start)，逐次增加，用DFS+剪枝替代BFS式扩展
        - 每次迭代用栈做DFS，只扩展f <= limit的节点
        - 若无解，limit = 本次搜索中最小的超出limit的f值
     b) 加权A*：f = g + w*h，(w=1.5~2.0)
        - 搜索更快但可能非最优
     c) 线性冲突：曼哈顿距离 + 2*同行同列冲突数
        - 仍是可采纳的，且比曼哈顿更精确

============================================
  创新方向（选1~3个深入做，PPT明确要求）
============================================

  ★ 算法层面创新

  ① 线性冲突启发式 (Linear Conflict)
     - 在曼哈顿距离基础上，检测同行/同列中两个数字互相阻挡的情况
     - 比如第1行是[3,1,0]，数字3和1都在目标行但位置颠倒
       → 至少需要多走2步 → h值+2
     - 仍然是可采纳的，且比普通曼哈顿更精确
     - 难度：★★☆☆☆ / 效果：扩展节点显著减少

  ② IDA* (Iterative Deepening A*)
     - 用递归+剪枝替代OPEN/CLOSED表
     - 优势：内存占用极小，适合困难用例
     - 劣势：可能重复扩展，耗时会增加
     - 难度：★★★☆☆ / 效果：解决A*内存爆炸问题

  ③ 加权A* (Weighted A*)
     - f(n) = g(n) + w × h(n)，w取1.2~2.0
     - w越大搜索越快，但路径可能不是最短
     - 对比不同w值的效果曲线
     - 难度：★☆☆☆☆ / 效果：速度提升明显

  ④ 双向A* (Bidirectional A*)
     - 从起点和目标各维护一个OPEN表
     - 当两个方向的搜索相遇时找到解
     - 理论扩展量减少一半
     - 难点：相遇条件和路径拼接
     - 难度：★★★★☆ / 效果：节点扩展数大幅减少

  ⑤ 模式数据库 (Pattern Database)
     - 将8个数码拆分为"Fringe Pattern"（1/2/3/4/5为一组）等
     - 预计算每个子模式的最优距离存入数据库
     - 取各组启发值的最大值作为最终h
     - 是学术界最强的可采纳启发式
     - 难度：★★★★★ / 效果：近乎完美的启发式精度

  ★ 交互/可视化层面创新

  ⑥ 搜索过程动画
     - 不只是显示最终路径
     - 用动画展示每一步棋盘滑动的过程（数字格子平移动画）
     - 颜色变化：当前移动的格子高亮
     - 支持调速（0.5x / 1x / 2x / 最大速度）

  ⑦ 多种输入方式
     - 键盘直接输入数字
     - 鼠标点击格子 + 数字面板填入
     - 一键随机生成（从目标出发随机打乱）
     - 预设经典关卡按钮（如"最难8数码"）

  ⑧ 搜索树可视化
     - 在GUI旁显示搜索树展开情况
     - 用节点颜色区分：绿色=已找到路径、蓝色=OPEN中、灰色=CLOSED
     - 显示实时统计：已扩展节点数、OPEN大小、当前最优f值

  ⑨ 启发式实时对比
     - 界面上同时显示3种启发式的搜索进度
     - 用进度条或柱状图展示扩展节点数的实时对比

  ★ 其他方向

  ⑩ 扩展到N×N（15数码/24数码）

     - 同样代码，改参数即可支持15数码(4×4)
     - 研究启发式在不同规模下的表现

  ⑪ Web在线部署
     - 用 HTML+CSS+JS 实现完整项目
     - 部署到 GitHub Pages，生成在线链接
     - 报告直接附链接，老师打开就能体验

  ⑫ 机器学习启发式
     - 训练简单神经网络预测剩余步数作为启发式
